## What Went Well

### 1. NFR6 enforced architecturally, three layers deep
- CLI-parse rejection: `--include-credentials` throws `SHARE_INVALID_SOURCE` before any read
- Decision-build redundant check: `buildBundle` called with hardcoded `includeCredentials: false`
- Profile-level unconditional strip: `applySanitization('strip-personal')` cannot preserve credentials
- TYPE-level enforcement: `SanitizationProfile`'s literal-string discriminator makes `credentials: 'preserve'` unrepresentable
- Mechanical verification: Story 4.3's byte-sweep test (`bundleText.includes('secret-test-token-EYES-ONLY')` → false)

### 2. Two long-deferred production bugs root-caused
- **Sessions-layout assumption** (alpha tester bug, 2026-05-12): Claude Code writes session JSONLs flat under `<slug>/*.jsonl`, not under `<slug>/sessions/`. The wrong assumption caused `fix-paths` to mis-decode any project with a hyphen in its name. Story 4.0 fixed reader, writer, and every test fixture in one pass.
- **Integrity-hash canonicalization gap** (Story 1.13 deferred D8, half-closed by Story 2.0): When `applySanitization` adds fields via object spread AFTER `buildBundle`'s Zod canonicalization, the serializer's hash was computed against pre-canonical bytes that didn't match the bytes Zod reconstructs on parse. Story 4.3's `serializeBundle` now re-canonicalizes inside the serializer itself — closes the complementary half of D8.

### 3. Declarative-data-as-contract pattern established
`SANITIZATION_PROFILES` is an inspectable data structure consumed by per-category helpers. The profile is reviewable as data; the helpers contain the per-category logic. Story 4.2's pattern overrides plugged into the same helper via a single optional parameter without changing the profile shape. **This is the pattern to repeat for any future security-sensitive policy.**

### 4. MEDIUM-finding count crashed from 10 → 1
The Epic 3 retro's "fail loudly before side effects" principle (carried forward as a standing AC pattern) shows up in every 4.x story's preconditions. The single MEDIUM that did surface (Story 4.1 `pathStartsWith` win32 mixed-separator gap) was a real edge case that the type system couldn't catch — exactly the kind of thing code review exists for.

### 5. 100% Epic 3 retro follow-through
Second epic in a row. The X.0 cleanup-before-feature-work pattern is now firmly established.

### 6. Pipeline resilience proven under real-world failure
The power-outage interruption during Story 4.0 CR killed the `cr-4-0` agent mid-spawn. Recovery: dev-4-0's work was preserved on disk; the Lead verified git state; a fresh `cr-4-0-resume` agent with explicit re-verify instructions ran the gates from scratch. No corruption, no rework, no lost story-file edits.

## Challenges and Lessons Learned

### 1. Bundle-format schema kept growing mid-epic
`sourceHomedir` (Story 4.1, required, drove version bump to 1.1.0), `profile` (Story 4.2, optional, no bump), `wasRedacted` (Story 4.1, optional). Each addition required fixture migration across `tests/fixtures/bundles/`. The bumps were architecturally correct, but Story 4.1's spec did not anticipate `sourceHomedir`'s necessity — it emerged when the dev realized the strip-personal profile needed a way to identify "absolute paths under the source machine's home directory." Lesson: when a profile's AC mentions a contextual fact (here, "source home directory"), the spec author should explicitly check whether that fact is already on the bundle or whether the schema needs to be extended.

### 2. Test-fixture infrastructure proliferated
Story 4.0 migrated every `seedClaudeTree` call site to flat layout. Story 4.3 added `seedShareSourceTree` AND extracted `snapshotTree` into a shared helper. The integration-test helper module is now the second-largest file under `tests/integration/helpers/`. Both moves were correct in scope, but watch this growth — at some point a Story 5.x cleanup pass should refactor for clarity.

### 3. Power-outage recovery is undocumented
The recovery flow (verify dev's work on disk via `git status`, check team config for stale member registrations, spawn a fresh agent under a new name with explicit "do not assume any prior work exists" instructions) worked on the first try, but it's only in this Lead's session history. The next interruption — whether power, network, or a Vitest worker pool deadlock — will require re-deriving the procedure. **Action item 1 captures this.**

### 4. Spec skeletons need adjustment for non-trivial pattern logic
Story 4.2's `composePatterns` spec proposed an exact-source match for exclude detection. The dev correctly identified this wouldn't work (a glob `todo*` becomes `/^todo.*/i` with source `^todo.*`, whereas the stock pattern is `/^todo/i` with source `^todo` — they're never equal). The dev chose `startsWith` matching. This is the second time in this epic (after Story 4.1's `pathStartsWith`) that the spec's implementation suggestion required mid-implementation correction. Lesson: for non-trivial pattern logic, the spec should provide design intent + a worked example, not a copy-paste skeleton. The skeleton looks authoritative even when it isn't.

## Key Insight

**Declarative-data-as-contract beats procedural-code-as-contract for security invariants.** The strip-personal profile's literal-type discriminator (`credentials: 'strip'`) makes NFR6 violations a compile-time error, not a runtime audit. Combined with the mechanical byte-sweep in the integration test, the invariant is proven at three independent layers. This pattern should generalize to any future security-sensitive policy. When Story 5.4 documents the bundle format, the docs should reference the profile constants directly so the doc and the code can't drift.
