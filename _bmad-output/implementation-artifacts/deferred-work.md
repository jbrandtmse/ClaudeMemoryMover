## Deferred from: code review of story-1.2 (2026-05-09)

All findings below are real enforcement gaps but were intentionally deferred because the Story 1.2 spec mandated *verbatim* "exact content" for each rule file; patching scope expansion in-place would violate the spec contract. Each item should be reconsidered as part of a future hardening story (candidate: a "rule hardening" story under Epic 1 cleanup, or rolled into the first story that touches each rule's call sites).

- **MEDIUM — `no-process-env-home` misses destructuring and computed-env access.** Bypass patterns: `const { HOME } = process.env;` walks past the `MemberExpression` visitor; `process['env'].HOME` is rejected because the rule requires `node.object.property.type === 'Identifier'` (computed access yields a `Literal`). Resolution path: add `VariableDeclarator` + `ObjectPattern` walker for destructuring and relax the env-access type check to accept the `'env'` `Literal` form. Add tests for both bypass patterns.
- **MEDIUM — `no-console-outside-output` does not block `console.info/debug/trace/table/dir`.** AC3 lists exactly `log/error/warn` plus the two `process.std*.write` forms, so the rule correctly matches AC text — but the architectural intent is that ALL stdout/stderr writes route through the Output module. Resolution path: expand `BANNED_IDENTIFIERS` to cover the full ban list, update AC text in a follow-up story, add tests.
- **LOW — `no-raw-json-parse` does not catch `JSON['parse'](text)` (computed property access).** Theoretical bypass; would only matter if a contributor deliberately works around the rule. Resolution path: relax the `callee.property.type === 'Identifier'` constraint to also accept a `'parse'` string `Literal`. Add a regression test.
- **LOW — `no-hardcoded-separator` does not catch template-literal separators (`` `${a}/${b}` ``).** Spec literally says "string literal whose value is exactly `'/'`", so template strings are out of scope by design. Resolution path: extend rule to walk `TemplateElement` cooked values when the next holes look path-shaped, or accept the gap as an acknowledged limitation in the rule's docs. Best paired with adoption of the path engine (Story 1.7+) so the gap is closed by call-site discipline rather than the lint rule.
