## Deferred from: code review of story-1.2 (2026-05-09)

All findings below are real enforcement gaps but were intentionally deferred because the Story 1.2 spec mandated *verbatim* "exact content" for each rule file; patching scope expansion in-place would violate the spec contract. Each item should be reconsidered as part of a future hardening story (candidate: a "rule hardening" story under Epic 1 cleanup, or rolled into the first story that touches each rule's call sites).

- **MEDIUM — `no-process-env-home` misses destructuring and computed-env access.** Bypass patterns: `const { HOME } = process.env;` walks past the `MemberExpression` visitor; `process['env'].HOME` is rejected because the rule requires `node.object.property.type === 'Identifier'` (computed access yields a `Literal`). Resolution path: add `VariableDeclarator` + `ObjectPattern` walker for destructuring and relax the env-access type check to accept the `'env'` `Literal` form. Add tests for both bypass patterns.
- **MEDIUM — `no-console-outside-output` does not block `console.info/debug/trace/table/dir`.** AC3 lists exactly `log/error/warn` plus the two `process.std*.write` forms, so the rule correctly matches AC text — but the architectural intent is that ALL stdout/stderr writes route through the Output module. Resolution path: expand `BANNED_IDENTIFIERS` to cover the full ban list, update AC text in a follow-up story, add tests.
- **LOW — `no-raw-json-parse` does not catch `JSON['parse'](text)` (computed property access).** Theoretical bypass; would only matter if a contributor deliberately works around the rule. Resolution path: relax the `callee.property.type === 'Identifier'` constraint to also accept a `'parse'` string `Literal`. Add a regression test.
- **LOW — `no-hardcoded-separator` does not catch template-literal separators (`` `${a}/${b}` ``).** Spec literally says "string literal whose value is exactly `'/'`", so template strings are out of scope by design. Resolution path: extend rule to walk `TemplateElement` cooked values when the next holes look path-shaped, or accept the gap as an acknowledged limitation in the rule's docs. Best paired with adoption of the path engine (Story 1.7+) so the gap is closed by call-site discipline rather than the lint rule.

## Deferred from: code review of story-1.4 (2026-05-09)

## Deferred from: code review of story-1.5 (2026-05-09)

- **MEDIUM — AC6 ESLint rule scope deviation: `src/**/*.test.ts` exempted in addition to the two service modules.** AC6 verbatim: "EXCEPT `src/services/write-gate.ts` and `src/services/backup-service.ts`." The dev added `src/**/*.test.ts` to the rule's `ignores` because tests need direct fs access for fixture setup and mock injection (`eslint.config.js:30`). Justification is documented in the story Completion Notes. Three independent test files in `src/core/` and `src/services/` already use `fs/promises` directly for setup. Resolution path: PM/Architect ratification — either confirm the exemption as intended (most likely; matches established project test pattern) and amend AC6 to explicitly allow `*.test.ts`, or split test fixtures into `tests/integration/` so the rule can apply strictly. No code action required pending the decision.
- **LOW-MEDIUM — AC4 strict-text deviation: `<claudeDir>/backups/**` is excluded from the recursive copy.** AC4 verbatim: "recursively copies the entire `claudeDir` tree." `stageClaudeCopy` filters out `<claudeDir>/backups` and any nested path under it (`backup-service.ts:18-25`) to prevent recursive backup-of-backups (combinatorial explosion). The deviation is necessary for correctness but is not in the spec. Resolution path: Architect updates AC4 (or the architecture's "Filesystem Discipline" section) to explicitly state that `backups/` is meta-state and excluded from new backups. No code action required.
- **LOW — backup-service.ts pruning may include non-backup entries that happen to live in `<claudeDir>/backups/cmemmov/`.** `readdir` returns ALL entries; `rm(..., { recursive: true })` removes them regardless of type. A stray `.DS_Store`, `Thumbs.db`, or accidental file in the backup root would be deleted. Resolution path: filter `entries` to only include directory names matching the timestamp pattern (`/^\d{4}-\d{2}-\d{2}T.*-\d+-[0-9a-f]{8}$/`) before pruning. Low risk because the directory is owned by cmemmov and shouldn't contain other entries; revisit if a user reports unexpected file deletion.
- **LOW — backup-service.ts cross-volume cp fallback can leak the staging directory.** If `rename(staging, backupDir)` rejects with EXDEV and the subsequent `cp(staging, backupDir, { recursive: true })` then throws (e.g., disk full mid-copy), `rm(staging, ...)` is not called and the staging dir orphans in `os.tmpdir()` (`backup-service.ts:50-52`). The OS cleans tmpdir periodically so this is bounded but not zero. Resolution path: wrap the cp call in try/finally so the staging dir is removed regardless of cp's outcome. Defer to a hardening pass — not a Story 1.5 blocker.
- **LOW — `recordedOps()` returns the live internal array reference.** The TypeScript `readonly WriteOp[]` annotation is compile-time only; callers can `as WriteOp[]` to mutate, breaking the gate's invariant that ops are immutable post-recording. Resolution path: return `Object.freeze([...ops])` or a `structuredClone(ops)` per call. Cost: per-call array copy. Defer until a real consumer demands runtime immutability — internal-only API for now.
- **LOW — `backup-service.test.ts:89-109` "does NOT prune when the new backup write throws" exercises an early-throw path (non-existent claudeDir → mkdtemp/cp throws), not a mid-function throw after the new backup is partially written.** The test demonstrates the property in spirit ("any throw before pruning short-circuits it"), but the trickiest variant — backup-write throws AFTER the rename but before pruning — is uncovered. Resolution path: add a fault-injection test using `vi.mock('node:fs/promises')` to force a mid-function throw on `readdir`, asserting that the just-created backup remains and prior entries are intact. Defer to a hardening pass.
