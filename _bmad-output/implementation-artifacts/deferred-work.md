- **LOW — `no-raw-json-parse` does not catch `JSON['parse'](text)` (computed property access).** Theoretical bypass; would only matter if a contributor deliberately works around the rule. Resolution path: relax the `callee.property.type === 'Identifier'` constraint to also accept a `'parse'` string `Literal`. Add a regression test.
- **LOW — `no-hardcoded-separator` does not catch template-literal separators (`` `${a}/${b}` ``).** Spec literally says "string literal whose value is exactly `'/'`", so template strings are out of scope by design.
