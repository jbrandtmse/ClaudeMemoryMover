## What Went Well

1. **ESLint architectural enforcement was highly effective.** Once Story 1.2 landed the four custom rules (`no-hardcoded-separator`, `no-console-outside-output`, `no-raw-json-parse`, `no-process-env-home`), subsequent stories achieved 90%+ first-try lint compliance. Drift that would have been painful on Windows CI was caught mechanically instead.

2. **Test suite compounded cleanly.** From placeholder tests in 1.1 to 337 passing tests across 27 files by 1.13. The `dry-run-isolation.test.ts` pattern became a reusable template. Very low friction adding new test files.

3. **Core user journey delivered.** Export → import → rollback works end-to-end on all three OSes. Alex can migrate her Claude setup today with `cmemmov`. That is the stated goal of Epic 1.

4. **Layered architecture held.** `core/` never leaked in the wrong direction. Discriminated unions on `ApplyCategoryOpts`, `ImportDecision`, `ExportDecision` prevented silent per-category handling bugs. Zero contract mismatches between `bundle-parser`, `bundle-serializer`, `claude-reader`, `claude-writer` after Story 1.6.

5. **Dynamic imports for startup budget met.** Lazy command loading achieved <500ms NFR1 on all three CI runners (median ~170ms on Windows). Verified by `npm run bench:startup`.

6. **Autonomous pipeline judgment was sound.** The Project Lead was largely tuned out during Epic 1 and retrospectively confirmed no decisions needed course-correction. The pipeline's defaults (defer LOW findings, auto-fix HIGH/MEDIUM in-context) were appropriate.

---

## Challenges

1. **Canonical key-order bug shipped undetected for 3 stories.** `buildBundle` constructs `Bundle` keys in author order; `BundleSchema.parse` reorders to schema-declaration order. The differing `JSON.stringify` outputs cause false `BUNDLE_CHECKSUM_MISMATCH` on any bundle with multiple optional keys under `global`. Was introduced in Story 1.10, not caught until Story 1.13's integration tests first exercised a multi-key global. All integration tests work around it with `integrityCheck: false`.

2. **`bundle.global.claudeJson` writer-surface gap.** Export bundles `~/.claude.json`; import ignores it (no `claudeJson` category in `ClaudeCategory` or `applyCategory`). A warn message was added as a mitigation, but this is a half-feature in production. Story 2.0 must close it before Epic 2 builds further on the import path.

3. **Stories 1.8 and 1.11 sat in review with no code reviewer spawned.** These were pre-pipeline stories (worked before the `/epic-cycle` automated loop started). They sat in `review` status for the duration of Epic 1 and weren't caught until the retrospective check. The pipeline handles new stories correctly; the gap was pre-existing state.

4. **deferred-work.md grew to ~115 entries with no triage owner.** Most are LOW, but several MEDIUMs are relevant to Epic 2. No prioritization pass happened during Epic 1.

---

## Key Insights

1. **Lint rules as living architecture docs.** Custom ESLint rules were more effective than documentation at preventing drift. The rules outlasted the stories that created them.

2. **Code review loop catches what tests miss.** The canonical key-order bug, the `claudeJson` drop, the bench-startup timeout silent-pass — all caught in code review, not by tests. Tests verify behavior; review verifies design intent.

3. **'Review' status with no assigned reviewer is a pipeline gap.** Add a check: if a story has been in `review` for more than one story cycle with no `cr-` agent spawned, surface a warning.

4. **Autonomous pipelines need a deferred-work triage ritual.** 115 deferrals with no owner creates hidden risk. A brief triage pass before each epic is now part of the lead's pre-epic checklist.
