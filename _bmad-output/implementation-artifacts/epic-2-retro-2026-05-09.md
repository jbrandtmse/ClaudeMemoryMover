## What Went Well

1. **Integration tests caught a production bug before shipping.** Story 2.4's cross-OS tests discovered that `resolveProjectsCrossOS`'s scripted `--remap` block preserved source-OS backslashes in the suffix, breaking `isInsideHome` on POSIX runtimes (`win→mac` and `win→linux` combinations would have silently failed in CI). The bug was found and fixed within the same story — zero user exposure. This validates the "test the central differentiator" goal of Story 2.4.

2. **Dev agent judgment improved.** `dev-2-4` proactively paused mid-implementation, flagged two issues (AC #3 slug encoding wording mismatch and the separator bug), and waited for direction rather than guessing. Both flags were correct. This prevented a round of rework that a "ship first, ask later" approach would have caused.

3. **Code review caught a HIGH that tests missed.** `cr-2-3` identified that `remapPermissionRules` only handled the flat-array `permissions: [...]` shape, not the real Claude Code `permissions: { allow, deny, ask }` nested object shape. Unit tests were written against the flat shape exclusively. The HIGH was patched in-context before commit. This is exactly the design-intent gap that tests alone cannot catch.

4. **100% follow-through on previous retro action items.** Story 2.0 was purpose-built to close Epic 1's deferred debt. All four action items were resolved before any Epic 2 feature work started. The pattern works.

5. **ESLint rules continued to enforce architectural discipline.** `no-hardcoded-separator` flagged literal `'/'` and `'\\'` in Stories 2.1 and 2.3 at lint time — caught before commit in both cases. The rules earn their keep every epic.

6. **`--remap` scripted mode enabled fully automated CI testing.** Designing cross-OS import with a `--remap "src=tgt"` flag (rather than purely interactive) was the key architectural choice that made Story 2.4's CI-portable integration tests possible. Scripted mode removes the need to mock interactive prompts for the six round-trip cases.

All were caught before commit, but the frequency suggests developers aren't running `npm run lint` between implementation steps. Not a quality gate failure, but a friction point.

## Key Insights

1. **Integration tests are the right place to find production bugs.** Unit tests verified individual function behavior; it was the full export→import→assert round-trip that exposed the separator normalization failure. For cross-platform code, end-to-end integration tests on real OS runners are not optional.

2. **Dev agent "pause-and-ask" is a feature, not a defect.** The pipeline design allows agents to escalate clarifications. When `dev-2-4` flagged the slug-encoding AC discrepancy and the separator bug before writing tests, it saved a full code review cycle. The overhead of the pause was less than the cost of rework.

3. **AC precision is a force multiplier.** Ambiguous or incorrect ACs (Story 2.4 slug encoding) cause mid-dev confusion and pipeline pauses. Time invested in precise ACs upstream saves disproportionate time downstream.

4. **Architecture docs must be treated as living code.** Two errors were found in `architecture.md` during Epic 2 (return type error in `suggestRemap`, other doc drift). These should be corrected before they propagate into Epic 3 story specs.
